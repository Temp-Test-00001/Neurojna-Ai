8. TECHNICAL ARCHITECTURE & DESIGN
--------------------------------------------------------------------------------
* Frontend Architecture:
  - Framework: React (TypeScript) + Vite
  - Styling: Tailwind CSS
  - Icons: Lucide React
  - Visualizations/Charts: Recharts & SVG custom designs
  - Client Routing: Simple conditional routing for `/admin` vs public page.
  - Interactive Canvas: Hero features a live particle-based neural network canvas simulating nodes and connecting lines reacting to mouse moves.
  - Theme Control: Context-based light/dark theme system.

* Backend Architecture:
  - Framework: Express.js (TypeScript)
  - Runtime: Node.js
  - Database Interface: raw `pg` client connecting to PostgreSQL.
  - Authentication: JSON Web Tokens (JWT) for admin sessions, with bcryptjs for hashing admin credentials. Expiration set to 24 hours.
  - Mailing System: Nodemailer configured with Gmail SMTP to trigger automatic email confirmations to visitors and notifications to the admin team upon contact submission.

* REST API Endpoints:
  - POST `/api/contact` (alias: `/api/enquiries`) - Public endpoint to submit contact enquiries. Saves to Database and sends notification and confirmation emails.
  - POST `/api/newsletter` (alias: `/api/subscribers`) - Public endpoint for waitlist/newsletter signup. Uses check-then-insert validation to avoid duplicates.
  - POST `/api/auth/login` - Public login endpoint returning JWT token.
  - GET `/api/admin/contacts` - Protected endpoint (requires JWT) listing all enquiries, ordered by date descending.
  - DELETE `/api/admin/contacts/:id` - Protected endpoint deleting a contact enquiry.
  - GET `/api/admin/newsletter` - Protected endpoint listing all newsletter subscribers, ordered by date descending.
  - DELETE `/api/admin/newsletter/:id` - Protected endpoint deleting a newsletter subscriber.
  - GET `/api/stats` - Protected endpoint yielding summary metrics (counts, charts data) for the Admin Dashboard.
  - GET `/api/health` - Health check.

* Database Tables Schema:
  - Table: `enquiries`
    * `id` SERIAL PRIMARY KEY
    * `name` TEXT NOT NULL
    * `email` TEXT NOT NULL
    * `company` TEXT
    * `service` TEXT
    * `message` TEXT NOT NULL
    * `status` TEXT DEFAULT 'new'
    * `created_at` TIMESTAMPTZ DEFAULT NOW()
  - Table: `subscribers`
    * `id` SERIAL PRIMARY KEY
    * `email` TEXT UNIQUE NOT NULL
    * `created_at` TIMESTAMPTZ DEFAULT NOW()
  - Table: `admins`
    * `id` SERIAL PRIMARY KEY
    * `username` TEXT UNIQUE NOT NULL
    * `password_hash` TEXT NOT NULL
    * `created_at` TIMESTAMPTZ DEFAULT NOW()

* Configuration & Environment Variables:
  - `DATABASE_URL`: Connection string for PostgreSQL.
  - `JWT_SECRET`: Secret key for JWT signing.
  - `GMAIL_USER`: SMTP user for sending emails.
  - `GMAIL_APP_PASSWORD`: SMTP app password.
  - `ADMIN_EMAIL` / `NOTIFY_EMAIL`: Email addresses targeted for admin alerts.
  - `ADMIN_USERNAME` / `ADMIN_PASSWORD`: Default admin credential seeds.
  - `PORT`: Server port.
